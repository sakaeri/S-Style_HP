# 株式会社S-Style コーポレートサイト 仕様書（Claude Code 開発用）

このドキュメントは、デザインプロトタイプ（`site/` 内の `.dc.html`）をもとに、Claude Code で本番サイトを実装するための仕様をまとめたものです。プロトタイプは静的な見た目の確認用で、本番はこの仕様に沿って実装してください。

- **会社名**：株式会社S-Style
- **事業**：Sランクキャディの育成・派遣。全国8拠点。契約ゴルフ場派遣／イベント（単発）派遣／プレーヤー指名に対応。
- **コンセプト**：「旅するキャディ」。マスコットは王冠をかぶったクイーンビー「BiBi」。

- **代表者**：代表取締役　榮 恵里香
- **本拠点**：千葉県（住所は未確定 → 要差し替え）
- **拠点**：本拠点＝千葉県／支部＝北海道・沖縄県・山梨県・兵庫県・大阪府・三重県・埼玉県（計8拠点）

### 推奨技術スタック
- **Next.js（App Router）+ TypeScript** を推奨（静的書き出し or SSR どちらでも可）。

## 4. お知らせ機能（旧「お役立ち記事」から変更）

### 要件
- 会社からの**お知らせ**を投稿・一覧・詳細表示できること。
- 例：「〇〇ゴルフ場との契約がスタートしました」「キャディ手配サービス『TeeRA』をリリースしました」「BiBiオリジナルグッズの販売を開始しました」。
- **カテゴリ**：`お知らせ` / `契約情報` / `サービス` / `グッズ` / `イベント`（拡張可）。カテゴリでフィルタできること。
- 一覧は日付降順。日付＋カテゴリバッジ＋タイトルの行レイアウト。ページネーションあり。
- 詳細ページはパンくず、カテゴリ、日付、タイトル、アイキャッチ、本文、一覧へ戻る、他のお知らせ。

### データモデル（例）
```ts
type NewsCategory = 'お知らせ' | '契約情報' | 'サービス' | 'グッズ' | 'イベント';

interface NewsPost {
  slug: string;          // URL用
  title: string;
  category: NewsCategory;
  publishedAt: string;   // ISO日付
  eyecatch?: string;     // 画像URL（任意）
  body: string;          // Markdown or リッチテキスト
  summary?: string;      // 一覧・OGP用
}
```

### 実装方針
- 運用者が非エンジニアの場合、**ヘッドレスCMS**（microCMS 等、日本語UI・国産で運用しやすい）推奨。`NewsPost` に対応するAPIスキーマを作成し、一覧/詳細/カテゴリ絞り込みを取得。
- もしくは Markdown ファイル（`content/news/*.md`）+ frontmatter による静的生成でも可。
- カテゴリバッジ色は宝石カラーをマッピング（契約情報=グリーン系、サービス=パープル系、グッズ=ゴールド×ローズ、イベント=ゴールド、お知らせ=グリーン）。

## 5. フォーム機能

3種のフォーム。いずれも**送信バックエンドが必要**（プロトタイプは `onsubmit="return false"` のダミー）。

### 5-1. 採用エントリー（recruit）
項目：お名前(必須) / フリガナ / メール(必須) / 電話 / 希望勤務地(8拠点+相談) / キャディ経験(未経験・経験あり) / メッセージ / 個人情報同意(必須)

### 5-2. ゴルフ場お問い合わせ（contact-course）
項目：ゴルフ場名(必須) / ご担当者名(必須) / 電話 / メール(必須) / ご相談内容(契約派遣・スポット派遣・料金条件・その他) / 詳細 / 同意(必須)

### 5-3. イベント/プレーヤー様（contact-event）
項目：ご依頼種類(必須:イベント派遣・プレーヤー指名・その他) / お名前(必須) / 電話 / メール(必須) / ご希望日 / プレー予定ゴルフ場名 / **事前確認済みチェック** / ご依頼内容 / 同意(必須)
- **重要**：プレーヤー指名は「事前にゴルフ場への問い合わせが必須」。注意書き（金枠の警告ボックス）を必ず表示し、確認チェックを設けること。

### 実装方針
- バリデーション（必須・メール形式）、送信後サンクス表示、自動返信メール、管理者通知。
- 送信先は要確認（メール転送 / CRM / スプレッドシート等）。スパム対策（reCAPTCHA等）推奨。
- フォームUIはすりガラスカード内。フォーカスリング `outline:2px solid #22564b`。

## 8. 非機能要件

- **レスポンシブ**：プロトタイプはPC幅中心。本番はモバイル対応必須（ナビはハンバーガー化、グリッドは1カラムへ、ヒット領域44px以上）。
- **SEO**：各ページ `<title>` / `<meta description>` 設定済み（プロトタイプ参照）。お知らせ詳細はOGP・構造化データ（Article/NewsArticle）を付与。サイトマップXML、robots.txt。
- **アクセシビリティ**：ガラス背景上のテキストはコントラスト確保（暗色オーバーレイ＋テキストシャドウ）。alt属性、フォームラベル、キーボード操作。
- **パフォーマンス**：`backdrop-filter` の多用に注意（モバイルGPU負荷）。画像は最適化（WebP/AVIF、遅延読み込み）。
- **ブラウザ対応**：`backdrop-filter` は `-webkit-` プレフィックス併記。

## 11. 管理ダッシュボード

`site/Dashboard.dc.html` に画面イメージ（プロトタイプ、値は保存されない）を用意済み。**本番ではこの5項目のみ**を非エンジニアが管理画面から更新できるようにする。それ以外の文言・レイアウト・デザインはコードで直接編集する運用でよい。

### 11-1. 対象の5項目
1. **拠点・契約先の管理**：拠点（都道府県名・本拠点/支店区分）の追加・削除、各拠点の契約ゴルフ場名の追加・編集・削除。6章の `Base` データ構造を編集するCRUD。反映先：ホームの拠点タグ、会社案内・採用情報の拠点一覧＋契約先ポップアップ（両ページで同一データソースを参照させ、連動させること）。
2. **お知らせの作成・管理**：4章の `NewsPost` のCRUD（日付・カテゴリ・タイトル・本文・アイキャッチ）。反映先：ホームの抜粋、お知らせ一覧、お知らせ詳細。
3. **お問い合わせ送信先の設定**：フォーム種別ごとに送信先メールアドレスを登録・変更できること。
   - ゴルフ場お問い合わせ（`contact-course`）→ 宛先A
   - イベントキャディお問い合わせ（`contact-event`）→ 宛先B
   - 採用エントリー（`recruit`）→ 宛先C
   各フォームの送信処理（5章）はこの設定値を参照して振り分ける。
4. **代表者写真の差し替え**：会社案内「代表挨拶」に使う横長写真をアップロードで差し替え。
5. （上記4項目に付随するデータの保存・反映が本番の主眼。それ以外の画像・文言差し替えはダッシュボード対象外＝コードで直接編集）

### 11-2. 実装方針
- **認証**：社内の担当者数名が使う想定。共有パスワード程度の簡易認証で可（要件に応じて強化）。
- **データ保存**：拠点・契約先／お知らせ／送信先設定は、CMS（microCMS等）またはDB（例：Supabase等）に保持し、各ページはビルド時 or リクエスト時に取得。写真はオブジェクトストレージ（S3等）にアップロードしURLを差し替え。
- **画面構成**：`Dashboard.dc.html` はタブ切り替え（拠点・契約先／お知らせ／お問い合わせ送信先／代表者写真）のUIをそのまま踏襲してよい。追加・削除・保存の操作感（インライン編集、行の＋/×ボタン）を参照。
- **バリデーション**：メールアドレス形式チェック（送信先設定）、必須項目チェック（お知らせのタイトル等）。

## 12. 移行の流れ（WordPress → 新サイト）

現在 WordPress.com でドメイン「onegolf.jp」を取得・管理中。WordPressの継続コスト（テーマ/プラグイン/ホスティング）を解消するため、以下の順で移行する。

1. **実装**：本SPECと `site/*.dc.html` を Claude Code に渡し、本番コードを実装（この間、現行WordPressサイトはそのまま公開され続けるため空白期間は発生しない）。
2. **デプロイ**：完成したコードを Vercel / Netlify / Cloudflare Pages 等の静的ホスティングへデプロイ（小規模サイトなら無料〜数百円/月）。デプロイ先の管理画面で独自ドメイン接続を開始し、指定される接続用レコード（Aレコード or CNAME、またはネームサーバー）を控える。
3. **動作確認**：デプロイ先が発行する仮URL（例：`xxx.vercel.app`）で本番相当の内容を一通り確認（全ページ・フォーム送信・お知らせ・ダッシュボード操作）。
4. **ドメイン切り替え**：WordPress.com にログイン →「ドメイン（Domains）」→「onegolf.jp」を選択 →「DNS設定」または「ネームサーバー」を、手順2で控えた新しい配信先の値に書き換える。反映まで数時間〜24時間。
   - **注意**：メール（`@onegolf.jp`）を運用中の場合、MXレコードは変更しない（サイト用のレコードのみ変更）。
5. **反映確認後、WordPressのホスティング/プラン契約を解約**してコスト削減完了。
